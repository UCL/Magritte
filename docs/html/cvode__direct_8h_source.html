59 /*-----------------------------------------------------------------
60  Type: CVDlsJacFn
61  -----------------------------------------------------------------
62 
63  A Jacobian approximation function Jac must be of type CVDlsJacFn.
64  Its parameters are:
65 
66  Jac is the SUNMatrix matrix that will be loaded by a CVDlsJacFn
67  with an approximation to the Jacobian
68  matrix J = (df_i/dy_j) at the point (t,y).
69 
70  t is the current value of the independent variable.
71 
72  y is the current value of the dependent variable vector,
73  namely the predicted value of y(t).
74 
75  fy is the vector f(t,y).
76 
77  user_data is a pointer to user data - the same as the user_data
78  parameter passed to CVodeSetUserdata.
79 
80  tmp1, tmp2, and tmp3 are pointers to memory allocated for
81  vectors of length N which can be used by a CVDlsJacFn
82  as temporary storage or work space.
83 
84  A CVDlsJacFn should return 0 if successful, a positive
85  value if a recoverable error occurred, and a negative value if
86  an unrecoverable error occurred.
87 
88  -----------------------------------------------------------------
89 
90  NOTE: See the relevant SUNMatrix implementation header files and
91  documentation for mechanisms to inquire about matrix
92  dimensions, and for efficient ways to set matrix entries.
93 
94  NOTE: If the user's Jacobian routine needs other quantities,
95  they are accessible as follows: hcur (the current stepsize)
96  and ewt (the error weight vector) are accessible through
97  CVodeGetCurrentStep and CVodeGetErrWeights, respectively
98  (see cvode.h). The unit roundoff is available as
99  UNIT_ROUNDOFF defined in sundials_types.h.
100 
101  -----------------------------------------------------------------*/
102 typedef int (*CVDlsJacFn)(realtype t, N_Vector y, N_Vector fy,
103  SUNMatrix Jac, void *user_data,
104  N_Vector tmp1, N_Vector tmp2, N_Vector tmp3);
